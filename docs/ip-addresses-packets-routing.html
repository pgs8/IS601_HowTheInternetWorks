IP Addresses, Packets, and Routing

 

An Internet Protocol (IP) Address is a unique address that identifies a device on a network. Billions of people use multiple devices each day to access the Internet. To keep track of all these devices and what they're doing, unique identifiers - IP Addresses - are necessary. An IP Address is like a mailing address for computers and other networked devices - if you know the IP address of a device, you can send information to it. Since IP addresses are hard to remember - after all they're just a string of numbers - domain names are nearly always used to send information.

Packets are formatted units of data containing header information and a payload sent from one device to another on a network. All information sent over the internet is split into packets. Each packet is a small (max 64kB) segments of a larger message, and contains a header and the payload. The header contains information about the packet itself - who sent it, where it is going, and where it falls in the order of the larger message. The payload is the information itself. As the packets are received, the destination device sends acknowledgements to the origin device, and then reassembles the packets to get the larger message. The origin device will resend any packets for which it does not receive an acknowledgement.

Routing refers to the path a packet takes from one device to another on a network. Each packet may take a different route from its origin to its destination. The routing is handled by specialized devices called routers, which refer to routing tables to determine which route to send packets. Routing tables can be static (manually updated) or dynamic (automatic).



 
 

Why do we use them?

Splitting larger messages into smaller packets to be sent over multiple routes from an origin device to the destination allows for networks to be distributed, decentralized, and highly fault tolerant.
