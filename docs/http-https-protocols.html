HTTP and HTTPS Protocols

 

Hypertext Transport Protocol (HTTP) is the protocol used to transfer data over the internet.

Hypertext Transport Protocol Secure (HTTPS) is the same as HTTP, but it uses a secure socket layer (SSL) for security.

 
 

Why do we use them?

HTTP is simple - messages can be read and understood by humans, making testing easier for developers, and reducing complexity for newcomers.

HTTP is extensible - HTTP headers make this protocol easy to extend and experiment with.

HTTP Request / Response communication between clients and servers is done by requests and responses:

  1. A client (a browser) sends an HTTP request to the web
  2. A web server receives the request
  3. The server runs an application to process the request
  4. The server returns an HTTP response (output) to the browser
  5. The client (the browser) receives the response
