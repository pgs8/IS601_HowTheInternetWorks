Domain Name System (DNS)

 

The Domain Name System is the address book for the internet. Humans enter an easy-to-remember domain name, and the DNS returns the computer-friendly IP Address. DNS is managed and coordinated by ICANN.

 
 

Domain Name System Example

Let's imagine David Lightman wants to view a webpage hosted by the server WOPR. Here's how that happens:

  1. David Lightman types WOPR into his browser.
  2. If he is using a modern web browser it will check its built in cache. If the IP Address for WOPR is found then the process is complete.
  3. If the browser came up empty, the next cache checked is the operating system.
  4. If the operating system also comes up empty, it will perform a DNS query on a Recursive DNS Server. Recursive DNS Servers are typically managed by an Internet Service Provider (ISP).
  5. The Recursive DNS Server first checks its local cache for the IP Address.
  6. If it does not have it cached, it will make a series of queries on David's PC's behalf to locate the Authoritative DNS Server that does have the IP address. This prevents David's PC from having to keep track of hundreds of thousands of Authoritative DNS Servers itself.
  7. Once the IP address is known, it is sent back to David's PC. The IP Address is also cached by the Recursive DNS Server, David's PC's operating system, and David's PC's web browser.
  8. David can now play a nice game of chess.

Why do we use it?

IP Addresses are hard to remember - they're just a string of numbers. Domain names on the other hand are mnemonic devices that easily identify a company, group, person, etc. Domain names are registered through any one of a number of ICANN accredited domain name registrars.
